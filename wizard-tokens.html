Token Proposal

Real OS UI HTML uses .wizard-wrapper + .list + [data-block="WizardItem"] + .wizard-wrapper-item (with .active, .past, .next state classes) + .wizard-item-icon-wrapper + .wizard-item-icon + .wizard-item-label. Proposal migrates raw color-neutral-* / color-primary references to semantic tokens, introduces a success semantic for the past state (completed ≠ just "reached"), and adds a disabled state absent from 2.28.

Property Utility class Current (2.28) Proposed Token Rationale
icon size .wizard-item-icon hardcoded 32px · glyph inherits 10px 36px · number 12px · icon glyph 14px --token-font-size-300 Grow to 36px. 32px is tight for a checkmark glyph — at 10px the ✓ reads as a pixel scribble. 36px gives the bubble enough room for a 14px checkmark (✓ reads as deliberate) without making the step numbers feel chunky. Empty-dot variant (8px) preserved for the compact progress-bead layout.
icon border .wizard-item-icon 1px solid color-neutral-5 (#ced4da) 2px solid token-border-default (#d5d5d5) --token-border-size-050
--token-border-default		 Thicker border (2px) so the active ring reads as a clear progress indicator — 1px disappears on high-DPI screens next to a 2px connector line.
connector .wizard-item-icon-wrapper::before 2px hardcoded · color-neutral-5 (next) · color-primary (past/active) 2px token · token-border-default · token-semantics-primary-base --token-border-size-050
--token-border-default
--token-semantics-primary-base		 Thickness unchanged (2px — a hairline track reads as lightweight progress, not a structural divider). The win is moving from raw color-neutral-5 / color-primary references to semantic tokens so dark-mode / brand-override themes inherit the connector palette consistently.
past state · colour .wizard-wrapper-item.past .wizard-item-icon bg color-primary (#1068eb) · number only bg token-semantics-primary-base (#105cef) · ✓ glyph --token-semantics-primary-base Past keeps the primary semantic (same family as active) — the past/active distinction is carried by fill vs. outline, not hue, so the wizard reads as one journey through a single brand colour instead of a traffic-light (green/blue/grey). Adds an explicit check glyph so completed steps are unambiguously done and not just "reached".
active state · ring .wizard-wrapper-item.active .wizard-item-icon border color-primary · no focus halo border + 4px primary 18% halo (keyboard-a11y) --token-semantics-primary-base (color-mix) Adds a subtle 18% primary halo around the active step so keyboard users landing on the wizard via tab-order can spot the current step without relying on the label colour alone. Halo sits outside the bubble so it doesn't shift the layout.
next state · icon text .wizard-wrapper-item.next .wizard-item-icon color-neutral-5 (#ced4da) — same as the border token-text-disabled (#a2a2a2) --token-text-disabled In 2.28 the next-step number uses color-neutral-5 — the SAME value as the icon border — so the digit fades into the circle outline and you can't read whether the step is "3" or "4" at a glance. Bump to text-disabled (#a2a2a2) for enough contrast to stay readable while still reading as "not yet active".
labels · base .wizard-item-label color-neutral-7 (#6a7178) — hardcoded palette step token-text-subtlest (#626262) --token-text-subtlest Same visual weight (low-emphasis label) but semantically anchored to text-subtlest — dark-mode themes and brand overrides get the right contrast flipped automatically instead of inheriting a fixed grey.
labels · state tokens .active / .past .wizard-item-label neutral-10 (active) · neutral-8 (past) text-default (active) · text-subtle (past) --token-text-default
--token-text-subtle		 Hierarchy unchanged (active = darkest, past = medium, next = subtlest) but backed by semantic text tokens so the whole wizard respects a single "text emphasis scale" instead of picking three arbitrary neutral steps.
disabled state .wizard-wrapper-item.disabled — (not in 2.28) icon + label → token-text-disabled · connector → token-border-subtle · opacity .6 --token-text-disabled
--token-border-subtle		 New state. Real 2.28 has no disabled wizard step — but flows with conditional branches (e.g. "billing only if paid plan") need a way to mark an upcoming step as locked/unavailable. Proposal introduces .disabled that tones down both the bubble and label without collapsing the step geometry.
vertical spacing .wizard-vertical .wizard-wrapper-item margin-bottom: var(--space-l) · raw alias margin-bottom: token-scale-600 (24px) --token-scale-600 2.28 uses the legacy --space-l alias. 2.29 already maps that to token-scale-600 — row is included for audit trail so theming overrides target the single token.
