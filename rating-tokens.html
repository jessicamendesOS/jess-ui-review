Token Proposal

Real OS UI HTML uses .rating + fieldset + .rating-item + 3 state divs (-filled, -half, -empty), coloured via .text-primary / .text-neutral-5 utility classes. Sizing is driven by --rating-size on the .rating container (8/16/24px). Icons use Font Awesome (.fa.fa-star/.fa-star-half).

Property Utility class Current (2.28) Proposed Token Rationale
color (filled) .text-primary (inside .rating) color-primary (#1068eb) — blue token-rating-filled-color (#e9a100) --token-rating-filled-color Neither 2.28 nor 2.29 gives Rating its own color — filled stars inherit primary blue from the .text-primary utility. Proposal introduces a dedicated rating-filled token tied to warning-base so stars read as an affective cue (gold) without hijacking the primary utility elsewhere.
color (half) .text-primary color-primary (#1068eb) token-rating-filled-color (#e9a100) --token-rating-filled-color Half star shares the filled color — same dedicated token keeps half/filled visually identical across theme overrides.
color (empty) .text-neutral-5 color-neutral-5 (#ced4da) token-border-default (#d5d5d5) --token-border-default In real 2.29 .text-neutral-5 is hardcoded (#ced4da). Promote it to token-border-default so empty stars share the low-emphasis gray used for dividers and input borders — one semantic token covers all three uses.
size --rating-size hardcoded 8px / 16px / 24px token-scale-200 / 400 / 600 --token-scale-200 / 400 / 600 Today each variant sets --rating-size to a raw pixel value. Proposal ties sizes to the spacing scale so Rating scales in lock-step with other components under density / responsive token sets.
item padding-right .rating-item calc(size + var(--space-s)) calc(size + token-scale-200) --token-scale-200 2.28 used the legacy --space-s alias. 2.29 already ties this to token-scale-200 — no palette mapping needed, but called out here for the audit trail.
disabled color .rating fieldset[disabled] text-color-neutral-6 · color-neutral-6 token-text-disabled (#a2a2a2) --token-text-disabled 2.29 already introduces --osui-rating-disabled-color as a semantic layer. Row is included for completeness so theming overrides can target this single token instead of the legacy color-neutral-6 palette.
