Token Proposal

Real OS UI HTML uses .rating + fieldset + .rating-item + 3 state divs (-filled, -half, -empty), coloured via .text-primary / .text-neutral-5 utility classes. Sizing is driven by --rating-size on the .rating container (8/16/24px). Icons use Font Awesome (.fa.fa-star/.fa-star-half).

Property Utility class Current (2.28) Proposed Token Rationale
color (filled) .text-primary (inside .rating) color-primary (#1068eb) — blue token-rating-filled-color (#e9a100) --token-rating-filled-color Neither 2.28 nor 2.29 gives Rating its own color — filled stars inherit primary blue from the .text-primary utility. Proposal introduces a dedicated rating-filled token tied to warning-base so stars read as an affective cue (gold) without hijacking the primary utility elsewhere.
color (half) .text-primary color-primary (#1068eb) token-rating-filled-color (#e9a100) --token-rating-filled-color Half star shares the filled color — same dedicated token keeps half/filled visually identical across theme overrides.
color (empty) .text-neutral-5 color-neutral-5 (#ced4da) — unrelated grey token-rating-empty-color — subtle tint of filled --token-rating-empty-color Real 2.28/2.29 render empty stars in a standalone neutral grey, disconnected from the filled color. Proposal derives the empty state from the filled token (color-mix(filled 38%, white)) so the widget reads as a single chromatic family — empty stars still clearly "off" but visually anchored to the rating color instead of looking like a third, unrelated tone. 38% (not 22%) keeps empty slots clearly visible so the row's visual rhythm reads as 5 stars at all times, not "3 stars and ghosts".
icon vertical-align .rating .icon baseline (Phosphor default) middle — Default vertical-align: baseline can subtly shift glyphs that have different metrics (e.g., ph-star vs ph-star-half) within a row. Forcing vertical-align: middle centers each glyph on the line-box midline so half/full/empty stars sit on a single optical centerline.
size --rating-size hardcoded 8px / 16px / 24px token-scale-300 / 400 / 600 (12 / 16 / 24px) --token-scale-300 / 400 / 600 Today each variant sets --rating-size to a raw pixel value. Proposal ties sizes to the spacing scale so Rating scales in lock-step with other components under density / responsive token sets. Small bumps from 8px → 12px: at 8px the star shape collapses into an illegible dot; 12px (the next step on the scale) keeps the compact footprint while preserving recognisable star geometry.
item padding-right .rating-item calc(size + var(--space-s)) calc(size + token-scale-200) --token-scale-200 2.28 used the legacy --space-s alias. 2.29 already ties this to token-scale-200 — no palette mapping needed, but called out here for the audit trail.
disabled color .rating fieldset[disabled] text-color-neutral-6 · color-neutral-6 token-text-disabled (#a2a2a2) --token-text-disabled 2.29 already introduces --osui-rating-disabled-color as a semantic layer. Row is included for completeness so theming overrides can target this single token instead of the legacy color-neutral-6 palette.
row label alignment .rating-row-label / .rating-row-item inherits text-align: center from #previewWrapper text-align: start — Preview wrapper centers all text by default; rating row labels were inheriting that and floating mid-column instead of sharing a left edge with the stars below. Adding .rating, .rating-row-item, .rating-row-label to the text-align: start allowlist anchors labels to the column's left edge so each row reads as a clean label → rating pair.
