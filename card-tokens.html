Token Proposal

Suggested changes to improve visual hierarchy and design system alignment.

Property Component Current (2.29) Proposed Token Rationale
border-radius .card, .card-background token-border-radius-200 (8px) token-border-radius-200 (8px) — no change --token-border-radius-200 Kept at 8px to match the Alert component and maintain a consistent container radius across the library.
border-color .card token-border-subtle (#f3f3f3) #e5e7eb (pending token: token-border-neutral-200) — The subtle border is nearly invisible on white. A slightly stronger border gives the card better definition without being heavy.
box-shadow .card token-elevation-1 0 1px 2px rgba(16,24,40,.04), 0 1px 3px rgba(16,24,40,.06) — Flatter shadow creates a more modern, border-forward appearance. Elevation can be reserved for interactive states (hover).
color .card-title inherited (no explicit token) token-text-default (#242424) --token-text-default Explicit token prevents unintentional color inheritance across theme overrides.
padding .card-image (card-sectioned) token-scale-600 (24px T/L/R) 0 --token-scale-0 Image flush to card edges creates a magazine-style layout consistent with modern card patterns.
border-radius .card-image img osui-card-border-radius (8px) 0 — Image no longer needs its own radius when flush — the card's overflow:hidden clips the top corners correctly.
background .card-bottom (card-sectioned) none (transparent) #fafbfc (pending token: token-bg-surface-subtle) — Subtle tint creates a visual separation between the content body and the action footer.
border-top .card-bottom (card-sectioned) none 1px solid #f3f4f6 — Explicit separator reinforces the body / footer distinction without adding visual weight.
width .card-bottom .btn auto (content width) 100% — Full-width CTA is a mobile-first best practice for primary actions inside card footers.
