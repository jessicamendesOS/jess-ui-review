Token Proposal

Suggested changes to improve visual hierarchy and design system alignment.

Property Component Current Proposed Token Rationale
border-radius .card, .card-background token-border-radius-200 (8px) token-border-radius-300 (12px) --token-border-radius-300 The Alert component already uses 12px. Aligning cards creates a consistent container radius across the component library.
border-color .card token-border-subtle (#f3f3f3) token-border-default (#d5d5d5) --token-border-default The subtle border is nearly invisible on white surfaces. token-border-default gives the card better definition and improves accessibility contrast.
box-shadow .card token-elevation-1 (very subtle) token-elevation-2 (stronger) --token-elevation-2 Elevation 1 is barely perceptible. A stronger shadow gives the card clearer visual hierarchy and helps it stand out from the page background.
color .card-title inherited (no explicit token) token-text-default (#242424) --token-text-default Explicitly setting the title color ensures correct rendering across theme overrides and prevents unintentional color inheritance.
