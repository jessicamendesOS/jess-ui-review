Token Proposal

Real OS UI HTML uses .btn + variant modifiers (.btn-primary, .btn-success, .btn-error, .btn-cancel) and size modifiers (.btn-small, .btn-large). Proposal shrinks default height (40→36px), softens the radius (4→8px), drops font-weight a notch (600→500), and replaces the filter: brightness hover with a cleaner shadow + ring.

font-weight.btnsemi-bold (600)medium (500) — uniform across all colored variants--token-font-weight-medium600 reads shouty next to body text. 500 keeps the CTA visually confident without over-competing with headings. Critical: the same weight applies to .btn-primary, .btn-success AND .btn-error — previously a stray Popup-section override was forcing the red error button to stay at 600 while siblings dropped to 500, making it look heavier than the rest. Now all primary-action variants render at a single weight; only .btn-cancel stays lighter as a secondary affordance.
