Token Proposal

Real OS UI HTML uses .breadcrumbs-content + .breadcrumbs-item + an .icon separator inside each item (except the last). Real 2.28 ships a minimal palette: all items render in color-neutral-8, the separator in color-neutral-7, space-s (8px) between items. Proposal migrates to semantic tokens, distinguishes the current page from its ancestors (2.28 makes no visual distinction), adds a chip-style hover, and introduces a collapsed overflow pattern for deep paths.

Property Utility class Current (2.28) Proposed Token Rationale
link colour .breadcrumbs-item a color-primary (#1068eb) — inherits global anchor token-text-subtlest (#626262) — explicit override --token-text-subtlest Real 2.28 lets breadcrumb links inherit the global primary-blue anchor colour. That's visually noisy — a 4-level breadcrumb ends up with three bright blue links competing with the actual page CTAs. Proposal quiets the chain by overriding to text-subtlest, pushing ancestors into the background so the current page becomes the only strong element.
current-page colour .breadcrumbs-content > .breadcrumbs-item:last-child,
.breadcrumbs-content > *:last-child > .breadcrumbs-item		 color-neutral-8 (#4f575e) — same grey as a span token-text-default (#242424) + semi-bold --token-text-default
--token-font-weight-semi-bold		 "You are here." 2.28 renders the current page in muted neutral-8 — so in a chain of blue links + grey span, the grey span reads as the weakest item despite being where the user actually is. Proposal lifts the current item to text-default + semi-bold so the active page becomes the strongest thing in the chain, and the ancestors (quieted to text-subtle) recede. The double selector handles both plain .breadcrumbs-item last children and wrapped structures (e.g. <li> wrappers) to avoid specificity conflicts with the font-weight regular on all items.
separator colour .breadcrumbs-item .icon color-neutral-7 (#6a7178) token-text-subtlest (#626262) --token-text-subtlest Slightly dimmer and semantically tokenised — the chevron reads as punctuation between interactive items rather than as content in its own right.
separator glyph .breadcrumbs-item .icon filled triangle (fa-caret-right / ph-fill @ 12px) outline chevron (Phosphor regular @ 12px) --token-font-size-300 Swap from filled triangle (▶) to thin outline chevron (›) — reads as punctuation between interactive items rather than a directional arrow. Same Phosphor codepoint \e13a; the override is purely a font-family swap from Phosphor-Fill to Phosphor, keeping the HTML shared with current-mode.
item font-weight .breadcrumbs-item inherits body (400) 400 regular — explicit --token-font-weight-regular Explicitly sets regular weight on all items so the semi-bold override on the last item creates a clear contrast without relying on inherited defaults that might vary by context.
hover state .breadcrumbs-item a:hover underline only (inherits link colour) colour shift only (text-subtlest → text-default) --token-text-default Drops the underline and the chip background entirely — hover just deepens the colour. Keeps the breadcrumb chain visually quiet and flat, matching modern tonal navigation patterns (GitHub, Linear, Notion all use this approach).
home-icon prefix .breadcrumbs-home — (not in 2.28; teams inline arbitrary icons) formalised as .breadcrumbs-home + ph-house — Home-icon breadcrumb starts are a universal pattern (GitHub, GitLab, Linear, every admin console). 2.28 has no formal slot, so teams reach for inline markup that renders inconsistently. Proposal introduces .breadcrumbs-home as the recognised first-item prefix so the icon stays aligned, weighted, and themable.
link transition .breadcrumbs-item a none color 100ms standard ease --token-duration-fast · --token-easing-standard Smooth colour fade on hover — prevents the abrupt jump from text-subtlest to text-default. Suppressed via prefers-reduced-motion.
link icon size .breadcrumbs-item a .icon inherits (variable) 1rem (16px) — explicit --token-font-size-400 Icons inside breadcrumb anchors (home icon, external-link indicator) are set to a consistent 16px so they sit optically aligned with the 14px text without growing too dominant.
collapsed overflow .breadcrumbs-item.breadcrumbs-collapsed — (not in 2.28) … item · hover chip · title-attr reveals ancestors --token-text-subtlest
--token-border-subtle		 New pattern. 2.28 has no convention for truncating deep paths — a 6-level breadcrumb on a narrow modal wraps onto two lines or pushes content out. Proposal introduces .breadcrumbs-collapsed as a single … item that groups middle ancestors. Hoverable (chip style) and ships with a title attr for the concatenated path; clicking can expand inline or open a dropdown (component-level decision).
