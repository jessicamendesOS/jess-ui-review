Token Proposal

Real OS UI 2.29 structure is just two classes: .progress-bar (outer track) > .progress (inner fill). There are no modifier classes for variants — consumers recolour the fill by overriding background-color on .progress.

Property Element Current (2.28) New tokens (2.29) Token Rationale
track background .progress-bar color-neutral-3 (#e9ecef) token-border-subtle → neutral-100 (#f3f3f3) --token-border-subtle Real 2.29 ships with a hardcoded primitive (neutral-300 #e0e0e0). Proposal: tokenise --trail-color to the semantic --token-border-subtle (neutral-100, lighter) so Progress Bar and Progress Circle share the same trail token — one decision point for both progress surfaces.
fill background (primary) .progress color-primary (#1068eb) token-semantics-primary-base (#105cef) --token-semantics-primary-base Fill participates in theme overrides with every other primary-coloured component. Real 2.29 uses nested fallback chain down to --token-primitives-blue-700.
fill background (success) .progress color-success (#29823b) token-semantics-success-base (#29823b) --token-semantics-success-base Same hex, now theme-aware. One decision point for progress + all other success-state components.
fill background (error) .progress color-error (#dc2020) token-semantics-error-base (#dc2020) --token-semantics-error-base Same hex, now theme-aware. Shared error token across destructive + error-state surfaces.
border-radius .progress-bar none (squared) token-border-radius-full (999px) --token-border-radius-full Real 2.28 ships square corners. Proposal: apply full-pill token consistently with buttons/badges/chips so the progress track feels native to the updated visual language.
height / thickness .progress-bar hardcoded px values token-scale-100 / 200 / 400 --token-scale-100 / -200 / -400 Thickness joins the spacing scale — scales proportionally with density adjustments rather than living as three magic numbers.
label (percent) .osui-progress-bar__content dark text on track (illegible over fill) Same as New tokens: token-text-default, justify-content: flex-end (no mix-blend) --token-text-default The New tokens bar keeps the value label end-aligned for consistency with 2.29; proposed matches that instead of inverse + mix-blend, which clashed with the same layout.
