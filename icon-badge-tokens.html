Token Proposal

Replace 2.28 palette vars in utility classes with 2.29 semantic tokens.

Property Component Current (2.28) Proposed Token Rationale
background .background-primary color-primary (#1068eb) token-semantics-primary-base (#105cef) --token-semantics-primary-base The utility class .background-primary should reference the semantic primary token so theming propagates to all components using this class without individual overrides.
background (error) .background-error color-red (#c92a2a) token-semantics-error-base (#dc2626) --token-semantics-error-base A dedicated error semantic token decouples the error state from the raw red palette, allowing the error color to evolve independently.
border-radius .border-radius-rounded border-radius-rounded (100px) token-border-radius-full (9999px) --token-border-radius-full 9999px is the standard full-pill value. Naming it token-border-radius-full makes the intent explicit and is consistent with Badge, Counter, and other pill-shaped components.
color (text) .badge (inside icon-badge) color-neutral-0 (#ffffff) token-text-inverse (#ffffff) --token-text-inverse token-text-inverse is the semantic token for text on colored backgrounds — correct for dark-mode and high-contrast overrides.
box-shadow (ring) .icon-badge .badge 0 0 0 2px #ffffff (hardcoded white) 0 0 0 2px token-surface-neutral-default --token-surface-neutral-default 2.28 paints a solid white ring to separate the badge from the icon underneath. Binding it to token-surface-neutral-default (the page background token) keeps the ring blending correctly in dark mode and themed surfaces instead of staying pure white.
