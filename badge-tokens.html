Token Proposal

Real OS UI HTML uses .badge + .background-{color} + .background-{color}-lightest + .text-{color}-darker + .border-radius-* + .OSInline utility classes. Proposal expands the palette with three new colors (purple, pink, indigo) and consolidates shape variants into sharp / soft / round, each backed by a border-radius token.

size (small) .badge-small font 12px · height 24px · min-w 24px · padding 0·4px font 10px · height 16px · min-w 16px · padding 0·4px --token-scale-100 (padding) Downsize for dense lists/sidebar counts — 24px was a chunky pill for a single digit. 16px height with 10px text still reads at normal viewing distance and tucks tighter against inline text.
size (default) .badge (no class) height 32px · min-w 32px · font inherited (≈16px) height 20px · min-w 20px · font 12px (token-font-size-300) --token-font-size-300
--token-font-weight-semi-bold		 Explicit font-size — 2.28 leaves the default badge font-inherited (renders at 16px), which was oversized for a count/status indicator and broke out of the circle at reduced heights. Lock it to token-font-size-300 (12px) and right-size height to 20px so a single digit renders as a clean compact circle.
size (medium) .badge-medium font 16px · height 40px · min-w 40px font 14px · height 28px · min-w 28px — Downsize — 40px was closer to avatar territory than a count badge. 28px keeps the "oversized" role for empty-state/hero banners but no longer dwarfs adjacent body text.
shape × size consistency .badge.border-radius-* sharp/soft inherit base 32px; only rounded was resized in preview prototypes all three shapes share the same height/min-width per size — Shape ≠ size. Shape modifiers (sharp / soft / round) only change border-radius — they must not affect badge dimensions. The proposed CSS applies the new heights at the base .badge level so the shape row renders as a pure corner-radius comparison and a .border-radius-rounded .badge-medium has the same footprint as a .border-radius-soft .badge-medium.
shape (sharp) .border-radius- — (2.28 uses .border-radius-none) radius 0 · bare class alias for .border-radius-none --token-border-radius-0 Class rename. 2.28 ships .border-radius-none; proposal keeps that utility and also accepts the bare .border-radius- suffix (seen in real OS UI HTML templates).
shape (soft) .border-radius-soft radius 4px (--border-radius-soft) radius 4px · (unchanged) --token-border-radius-100 Real OS UI utility — semantic soft-corner. Retarget to the 2.29 radius-100 token so all "soft" surfaces share one value.
shape (round) .border-radius-rounded radius 100px · always pill/ellipse radius 999px · circle for 1 char, pill for 2+ --token-border-radius-full Shape fix. Round badge with a single character now renders as a perfect circle (height = min-width). Multi-character content (e.g. "99+") still expands to a pill. Token value 999px in 2.29 (from 100px in 2.28).
