Proposed changes — Alert · based on M3 + Radix

CSS-only. HTML structure unchanged. Uses existing tokens or overridden token defaults.

Property Current Proposed Token Rationale
outer border 1px solid removed token-border-size-025 → 0 Tonal background provides sufficient visual containment without a stroke
border-left-width (accent) 3px removed (0px) — Left accent removed — background alone communicates the semantic variant
border-radius 8px 8px token-border-radius-200 Current value retained — already appropriate for inline feedback messages
gap (icon ↔ message) 16px (icon margin-right) 8px (flex gap) token-scale-200 Tighter coupling between icon and message — current 16px reads as disconnected
icon size 24×24 20×20 — Smaller icon matches the reduced text size and balances optical weight
padding 16px all sides 12px 16px (tighter vertical) --token-scale-300 · --token-scale-400 Reduced vertical padding makes alerts more compact and less heavy on the page
icon align-self flex-start flex-start (unchanged) — Icon stays top-aligned to anchor to the first line of text in multi-line messages
message font-size 16px 14px (0.875rem) --token-font-size-350 Smaller text gives alerts a compact, informational feel consistent with common web patterns
message font-weight 400 500 --token-font-weight-medium Medium adds emphasis without the heaviness of semi-bold; pairs better with 14px size
transition none background-color · color · border-color — 120ms standard ease --token-duration-fast · --token-easing-standard Smooth colour change when the variant type or theme changes at runtime, avoiding a jarring instant swap
