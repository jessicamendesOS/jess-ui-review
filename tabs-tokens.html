No visual change vs 2.28 (#4f575e → #626262 is within ΔE ≈ 3) but theming-ready: the single semantic token drives inactive tabs, secondary captions, and breadcrumbs.
emphasis .osui-tabs__header-item.osui-tabs--is-active font-weight: 600 (caused reflow) text-shadow: 0 0 0.5px currentColor — 2.29 already replaces the weight-switch with a sub-pixel text-shadow, which emphasises the active label without triggering 1-2px layout shift when changing tabs. Row is listed for the migration audit trail.
hover background .osui-tabs__header-item:hover none rgba(0,0,0,.04) + radius-100 pending: --token-bg-surface-hover Neither 2.28 nor 2.29 gives inactive tabs a hover background — the widget relies on color change alone. Proposal adds a subtle fill + top-radius for affordance, matching button / list-item patterns.
hover text colour .osui-tabs__header-item:hover:not(.is-active) text-subtle → active colour (primary) token-text-default (black) --token-text-default Real 2.28/2.29 hover promotes text to the active colour, which in the proposed scheme is primary blue — making a mere hover look like an activation. Proposal routes hover to text-default so primary stays reserved for the truly active tab.
separator color .osui-tabs__content color-neutral-5 (#ced4da) token-border-subtle (#f3f3f3) --osui-tabs-border-color 2.29 already promotes the separator to --osui-tabs-border-color (default: token-border-default #d5d5d5). Proposal re-points that variable to token-border-subtle so the active indicator is the dominant cue.
