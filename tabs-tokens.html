Suggested changes to improve visual clarity and semantic token alignment.

.osui-tabs--is-active		 text-color-neutral-10 (#101213) token-semantics-primary-base (#105cef) --token-semantics-primary-base The active tab uses the same color as the indicator — using the primary semantic token makes this relationship explicit and supports theming without two separate overrides.
color (inactive) .osui-tabs__header-item text-color-neutral-8 (#4f575e) token-text-subtle (#6b7280) --token-text-subtle The 2.28 neutral-8 is darker than needed — token-text-subtle is the standard secondary text color, creating the right contrast hierarchy against the active tab without being too heavy.
height .osui-tabs__header__indicator 2px (hardcoded) 3px (hardcoded) — A 3px indicator is more visible at desktop densities and matches common design system conventions (Material, Fluent, Carbon). Still thin enough not to feel heavy.
border-radius .osui-tabs__header__indicator none 3px 3px 0 0 — Slight rounding on the indicator top corners softens the edge without breaking the flush-to-border-top appearance. Complements the increased height.
hover background .osui-tabs__header-item:hover none rgba(0,0,0,.04) pending: --token-bg-surface-hover No hover feedback makes inactive tabs feel unresponsive. A subtle background fill on hover provides clear affordance that tabs are interactive, matching button and list-item patterns.
