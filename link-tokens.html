Token Proposal

Real OS UI uses <a data-link> combined with utility classes (.text-primary, .heading6). Proposal adds a subtle underline at rest (inline-link accessibility win) and formalises the focus-visible outline.

PropertyCurrent (2.28)ProposedTokenRationale
default underlinenone (underline only on hover)35% opacity underline at rest · full on hover—A11y win. WCAG 2.2 recommends another indicator besides colour for inline links. A subtle 35%-opacity underline at rest gives that indicator without dominating body text; hover deepens it to full opacity for feedback. Colour-blind users can now distinguish links from surrounding prose.
text-underline-offsetbrowser default (0)3px—Prevents the underline from clipping descenders (g, p, q) and keeps the typographic rhythm clean.
