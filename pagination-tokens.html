Token Proposal

Real OS UI HTML uses .pagination + .pagination-counter + .pagination-container + .pagination-button (with state modifiers .is--active, .is--ellipsis, and native [disabled]). Proposal drops the border grid on rest-state buttons (cleaner chrome), promotes the active state to a solid-filled primary chip, bumps the button size to 36px, and migrates palette references to semantic tokens.

Property Utility class Current (2.28) Proposed Token Rationale
button size .pagination-button 32px · 32px 36px · 36px (token-scale-900) --token-scale-900 32px is below the 44px mobile tap-target minimum and reads as cramped on desktop next to chunky content rows. 36px adds 4px of breathing room, puts each button closer to accessibility guidance, and still keeps a 10-button pagination narrow enough for a modal.
button border .pagination-button 1px solid color-neutral-4 (#dee2e6) none (rest) · only active carries colour — Drop the border grid. 10 bordered squares in a row reads as visual noise — the borders compete with the digits they contain. Proposal removes borders on rest-state buttons; hover surfaces them with a subtle background, and the active state carries the only coloured fill in the row. Result: the chain of digits reads cleanly as one unit, with "you are here" unambiguously strongest.
border-radius .pagination-button 4px (border-radius-soft) 8px (token-border-radius-200) --token-border-radius-200 Softer chip corner matches the card / popover treatment used elsewhere in 2.29. 4px reads "button-y"; 8px reads "pill-y" and pairs better with the borderless rest state.
rest colour .pagination-button color-neutral-8 (#4f575e) token-text-default (#242424) --token-text-default Since rest-state is borderless, text needs to hold its own as the only affordance — text-default (near-black) ensures digits read cleanly at 13px. All items share this colour so the row reads as one unit, distinguished only by the active chip's outline.
rest weight .pagination-button inherited (400) token-font-weight-medium (500) --token-font-weight-medium Digits need slightly more weight to hold their own at 13px without a border — 500 reads crisp without pushing into "strong CTA" territory (600+).
active state .pagination-button.is--active 1px primary border chip + primary text · white bg 1px neutral border · token-text-default · same weight --token-border-default
--token-text-default		 Shape, not colour. Active is carried by a thin grey border chip around the same dark text — every item shares the same tone, so the row reads as one uniform control rather than a grid of states. Matches modern minimal pagination patterns (Tailwind UI, Vercel admin, Plain.com) where the active page is a quiet outline rather than a shouty primary fill.
hover state .pagination-button:hover rgba(21,24,26,0.04) bg · border unchanged token-border-subtle bg · token-text-default color --token-border-subtle
--token-text-default		 Since rest-state buttons are borderless, hover needs to surface the button as a chip. border-subtle (#f3f3f3) is just dark enough to lift the digit out without competing with the active primary fill.
button spacing .pagination-button margin-left 8px (space-s) 4px (token-scale-100) --token-scale-100 With borderless chips and a solid-filled active state, 8px gaps make the row feel stretched out. 4px closes the rhythm so the chain reads as one connected control instead of 10 separate buttons — and the active chip's fill contrast is enough to keep it visually separated from its neighbours.
go-to-page input .pagination-input · .form-control[data-input] 32×32 · neutral-4 border · radius 4px · focus: browser default 36×36 · border-default · radius 8px · focus: single primary border (no outline ring) --token-border-default
--token-border-radius-200
--token-semantics-primary-base		 Aligns the input with the proposed chip aesthetic (same thin border, same radius-200, same text-default colour). Focus swaps the existing border colour to primary — no outline ring on top of the border, to avoid the doubled-chrome artefact of two concentric lines.
ellipsis .pagination-button.is--ellipsis 16px · transparent bg · no border auto-width plain dots · 8px padding · no chip --token-scale-200
--token-text-subtlest		 Unchanged in spirit — the ellipsis stays as plain dots, but sizes fluidly to content instead of being pinned at 16px. Colour migrates to text-subtlest so the dots read as passive punctuation between the numbered chips.
