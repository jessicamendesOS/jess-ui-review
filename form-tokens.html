Token Proposal

Real OS UI uses <form data-form class="form card"> with .columns.columns2 for 2-column layouts, [data-label] + .mandatory for required fields, and .input-text / .input-email around .form-control[data-input]. Proposed in this lab matches the user-profile reference in visual styling (same primary + radii as the component token labs, 1px field borders from --token-border-input-default, 36px control height, focus ring): no card chrome on the form, 24px row rhythm, bold 14px labels, optional trailing red affix aligned to the input height, and the same component rules as Input, Dropdown, Radio, Textarea, Switch, Checkbox, and Button in os-ui-229-proposed-scoped.css (same --token-border-input-default chain as the component labs).
