Token Proposal

Real OS UI uses .popup-backdrop + .popup-dialog + .popup-content + .heading3. Proposal softens the dialog corner (4→12px), deepens the elevation, and adds a backdrop blur for a modern feel.

PropertyCurrent (2.28)ProposedTokenRationale
border-radius4px12px--token-border-radius-300Modern modal pattern uses larger radius to distinguish floating surfaces from inline form fields. 12px reads as "overlay", 4px reads as "form element".
elevationsubtle shadowdeeper 2-layer shadow (24px · 8px)—Dialogs need real depth to assert themselves as overlay surfaces — a subtle shadow gets lost behind the backdrop dim.
backdroprgba(0,0,0,0.5) — flatrgba(15,23,42,0.5) + 4px blur—Slate-tinted backdrop reads warmer than pure black. The 4px blur softens background UI so the dialog doesn't fight with visible page content.
actions layoutright-aligned (inline style)right-aligned via flex--token-scale-200Formalise the action slot — 8px gap between buttons, right-aligned. Works with any button count (1-3 realistically).
close button<a> with fa-times icon<button> with ph-x + hover bg--token-border-subtleSemantic <button> with keyboard-accessible hover chip — <a href="#"> for a close action is an a11y anti-pattern.
