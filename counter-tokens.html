Token Proposal

Replace hardcoded variant colors with semantic base tokens; reference scale tokens for size dimensions.

Property Component Current (2.28) Proposed Token Rationale
background (error) .osui-counter--error hardcoded #dc2626 (was color-red) token-semantics-error-base (#dc2626) --token-semantics-error-base (pending) A dedicated semantic token means the error red is defined once and shared across Counter, Badge, Alert, and Toast — no per-component hex overrides needed.
background (warning) .osui-counter--warning hardcoded #d97706 (was color-orange) token-semantics-warning-base (#d97706) --token-semantics-warning-base (pending) The 2.28 orange palette covered both warning and orange-brand uses. A semantic warning-base token separates those concerns cleanly.
background (success) .osui-counter--success hardcoded #16a34a (was color-green) token-semantics-success-base (#16a34a) --token-semantics-success-base (pending) Matches the token already used by Progress and Status indicators — one success green across the entire system, easily updated via a single token override.
min-width / height (small) .osui-counter--small hardcoded 16px token-scale-400 (16px) --token-scale-400 Anchoring size to the scale system means any global density change (compact/cozy mode) propagates automatically to all counter sizes without touching component CSS.
min-width / height (large) .osui-counter--large hardcoded 28px token-scale-700 (28px) --token-scale-700 28px sits between scale-600 (24px) and scale-800 (32px) — a token-scale-700 step should be added to the scale to cover this size without a magic number.
border-radius .osui-counter border-radius-rounded (100px) token-border-radius-full (9999px) — no visual change --token-border-radius-full Semantic token name explicitly communicates intent (full pill), and is consistent with Badge — both pill components share the same radius token.
color (text) .osui-counter color-neutral-0 (#ffffff) token-text-inverse (#ffffff) --token-text-inverse token-text-inverse is the canonical white text token — semantically correct for text on colored backgrounds, and respects dark-mode or high-contrast overrides.
